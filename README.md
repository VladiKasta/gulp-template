# QScape 
документация по работе со сборкой

### *Установка производится путём клонирования -*

```git clone https://github.com/VladiKasta/gulp-template.git```

При настраивании нового проекта меняйте название в файлах package.json и package-lock.json

## 0. Команды
    - ```gulp``` (запускает локальную версию проекта)
    - ```gulp build``` (сбилживает файлы в папку dist)
    - ```npM run lint``` (посмотреть свои ошибки в коде)
    - ```npM run lint:fix``` (автоматически пофиксить ошибки)

## 1. Расположение файлов:
   
   ### SCSS

   src/

    ├──  scss/
    ├── pages-style/
    │ ├── index.scss # Главная страница + импорты всех SCSS файлов
    │ └── *.scss # Файлы стилей для конкретных страниц
    ├── global.scss # Глобальные стили (html, body, переиспользуемые классы)
    ├── media.scss # Брейкпоинты (от большего к меньшему) 
    └── vars.scss # Переменные для переиспользования

   ### HTML

   src/
   
    ├── partials/
    │ └── *.html # Переиспользуемые части приложения
    └── index.html # Главная страница

   ### JS
   src/
   
    └── js/
    ├── animate/
    │ └── animate-*.js # Анимации (не связанные с бизнес-логикой)
    ├── *.js # Дополнительные JS файлы
    └── script.js # Главный файл для импорта и использования функций

   ### РЕСУРСЫ
   src/
   
    ├── fonts/
    │ └── *.woff2 # Файлы шрифтов (woff2 или ttf)
    ├── icons/
    │ └── *.woff2 # Файлы иконок (woff2 или ttf)
    └── img/
    └── */ # Подгруппы изображений
    └── *.img # Файлы изображений

   ### СБОРКА
      -- dist (запускается командой gulp build и хранит в себе все минифицированные файлы)

   ### КОМПИЛЯЦИЯ В СБОРКУ
      -- gulpfile.js (хранит в себе функции, которые компилируют сборку при командах "gulp" "gulp-build")

   ### УПРАВЛЕНИИ ЗАВИСИМОСТЯМИ ПРОЕКТА
    -- package.json (хранит в себе описание проекта и его зависимостей)
      
    -- package-lock.json (хранит в себе точные версии и их транзитивные зависимости)

   ### GIT
    - gitignore (хранит в себе все файлы, которые не будет попадать и просматриваться гит)

## 2. Название сущностей:

    - Название папок/файлов должно быть в нотации kebab-case.
      Все папки должны быть названы подстраиваясь под их дочерние элементы.

    - Название классов должно быть по методологии BEM через тире и нижнее подчёркивание.
      Разбивайте класс по смыслу:
      header(класс родитель),
      header__container(контейнер класса родителя)
      header__nav-item (в родители создаём подблок nav и прописываем то, что лежит в этом блоке. В данном случае - item)
      Подробнее про данную методологию можно прочитать в подробке полезных ссылок в Доске документации

    - Название для глобальных переменных должно быть в нотации camelCase

    - Название функций должно быть в camelCase

## 3. Импортирование файлов
    - Переиспользование html идёт из папки partials по пути @@include('partials/header.html')

    - Когда вы написали свою функцию, её нужно импортировать и вызвать в файле script.js

    - Для импортирования scss файлов используйте новый формат @use '' as *;
